# 版本号递增 & CHANGELOG 更新

该技能用于在每次修改 Mod 后，自动递增版本号并记录修改内容。

## 触发时机

用户说 "bump version"、"更新版本"、"发版"、"记录修改" 等。

## 执行流程

### 第 1 步：确定变更类型

询问用户本次修改属于哪种类型：

- **功能新增/需求变更** → 递增**中间位**（1.13.6 → 1.14.0，末位归零）
- **Bug 修复** → 递增**末位**（1.13.6 → 1.13.7）

如果用户已经明确说了类型，跳过此步。

### 第 2 步：读取当前版本

读取 `modinfo.lua`，找到 `version = "X.Y.Z"` 这一行，解析出三个数字。

### 第 3 步：计算新版本

- 功能新增：`X.(Y+1).0`
- Bug 修复：`X.Y.(Z+1)`

### 第 4 步：收集修改内容

询问用户本次修改的具体内容。收集完毕后，汇总为一个清晰的分类列表：

```
### 新增
- ...

### 修改
- ...

### 修复
- ...

### 优化
- ...
```

### 第 5 步：更新 modinfo.lua

使用 Edit 工具将 `version = "旧版本"` 替换为 `version = "新版本"`。

### 第 6 步：更新 CHANGELOG.md

在 `docs/CHANGELOG.md` 中找到 `# 小月亮 (Little Moon) 修改记录` 标题后的第一个 `##` 条目之前，插入新的版本记录条目：

```markdown
## 新版本 - YYYY-MM-DD

<用户提供的修改内容>
```

使用 Edit 工具，在 `# 小月亮 (Little Moon) 修改记录\n\n` 之后插入新条目。

### 第 7 步：确认

向用户展示：
- 旧版本 → 新版本
- 新增的 CHANGELOG 条目内容
- 确认两个文件已更新
